# Version Update Log

| 日期 | 修改內容名稱 | 修改內容大綱 | 修改者 | 註記 |
|---|---|---|---|---|
| 2026-06-12 | v1.6.0 語音修復 + 安全強化 | Whisper STT server 因 shared library 遺失 + LD_LIBRARY_PATH 未設而靜默失敗 687 次；重新 build whisper.cpp、補上 libwhisper.so/libggml.so、service 加 LD_LIBRARY_PATH。同時新增 PID lock 防 duplicate instance、TokenFilter 遮 bot token log。opencode_client 從 opencode serve 改 direct Zen API。 | claudeyw | **Root Cause:** 之前 build whisper-server 只複製 binary 忘了裝 shared library，且 systemd user service 無 `LD_LIBRARY_PATH` 導致 `status=127` 無限 restart loop 但無 monitoring。**改進:** 每次 build 後必須 `ldd` 驗證 binary 完整性；user service 需設 `Environment=` 傳遞 lib path。 |
| 2026-06-09 | v1.4.0 Connection Pool | opencode_client.py 改用共用 httpx.AsyncClient（max 50 connections, keepalive 30s），取代每次 new client。新增 close_client() 優雅關閉。 | claudeyw | **Root Cause:** 每個 chat/TTS 請求都建立新 httpx client → 每次建立新 TCP 連線 → 增加延遲 + 浪費 socket。**建議:** 共用 client 需注意 thread safety（asyncio 單執行緒無此問題）。若連線數不足，調高 httpx.Limits(max_connections)。 |
| 2026-06-09 | v1.3.0 Session Timeout Fix | opencode_client.py httpx timeout 從 30s 提高至 120s。 | claudeyw | **Root Cause:** OpenCode 首次 session 初始化需下載 provider model + 建立 MCP 連線，耗時超過 30s。**建議:** 若經常有 timeout，可在 opencode.service 預先 warm up session（POST /session）。 |
| 2026-06-08 | v1.2.0 OpenCode Serve API | 從直接呼叫 NVIDIA NIM API 改為 opencode serve HTTP API（POST /session + POST /session/{id}/message），讓 Telegram Bot 與 CLI 使用相同的 agent、config、AGENTS.md、工具。 | claudeyw | **Root Cause:** 直接呼叫 LLM API 無法使用 OpenCode 的 AGENTS.md 系統提示、工具呼叫、MCP 整合，導致 bot 回答不如 CLI 品質。**建議:** opencode serve 必須先啟動（systemctl --user start opencode）。API key 透過 EnvironmentFile 餵給 serve daemon。若 serve 掛掉，bot 需有退避重試邏輯。 |
| 2026-06-08 | v1.1.0 DeepSeek + Rate Limit | 模型從 Nemotron 3 Super 切換至 deepseek-ai/deepseek-v4-flash。精簡 system prompt（只保留核心身份），改為讀取 AGENTS.md + memory_compact.md 作為系統提示。 | claudeyw | **Root Cause:** Nemotron 3 Super（120B）回應速度過慢。system prompt 包含所有 identity 規則導致 token 過長，觸發 NVIDIA NIM timeout。**建議:** 模型優先選 deepseek-v4-flash（速度快）。prompt 長度需監控，若持續 timeout 則進一步縮減 AGENTS.md。 |
| 2026-06-08 | v1.0.0 Initial Release | 首次發布：Telegram 語音/文字對話 Bot。支援語音 → Whisper STT → LLM → TTS 回覆、文字對話、白名單、systemd。使用 NVIDIA NIM API + edge-tts。 | claudeyw | **Root Cause:** 從零建立。選擇 python-telegram-bot v21（async native）而非 python-telegram-bot v13（sync callback）。選擇 edge-tts（免費、無 API key、中文品質好）而非 Azure TTS。**建議:** 若 edge-tts 延遲過高，可考慮切換至 Coqui TTS 或 OpenAI TTS。env 檔案務必加入 .gitignore。 |
